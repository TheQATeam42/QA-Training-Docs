Testing Talks React App - Contacts & Playground

גרסה 1.0 ינואר 2026 Testing Community

1. סקירה כללית

אפליקציית Testing Talks React App היא אפליקציית React פשוטה המיועדת לצרכי הדגמה ולמידה. האפליקציה מורכבת משני מודולים עיקריים:

📇

מערכת ניהול אנשי קשר

מערכת CRUD מלאה לניהול רשימת אנשי קשר עם יכולות חיפוש, יצירה, עריכה ומחיקה.

🎮

דף Playground

דף הדגמה המציג מגוון אלמנטי HTML ורכיבי ממשק משתמש לצרכי בדיקה ולמידה.

1.1 מטרת האפליקציה

האפליקציה משמשת כדוגמה לימודית וכלי לבדיקות עבור מפתחים ובודקי תוכנה:

  • הדגמת יכולות React בסיסיות
  • תרגול פיתוח ובדיקה של ממשק משתמש
  • סביבה בטוחה לתרגול כלי בדיקה אוטומטיים
  • דוגמה למבנה פרויקט React נקי ומסודר

2. מודול ניהול אנשי קשר (Contacts)

2.1 תצוגת רשימת אנשי קשר

תצוגה ראשית - רשימת אנשי קשר

תיאור תצוגתי:

  • טבלה/רשימה מסודרת של כל אנשי הקשר
  • כל איש קשר מוצג עם: שם, טלפון, אימייל, כתובת
  • כפתורים לכל איש קשר: "ערוך" ו-"מחק"
  • כפתור ראשי "הוסף איש קשר חדש"
  • שדה חיפוש בראש הדף

דרישות פונקציונליות:

  • הצגת כל אנשי הקשר בטבלה מסודרת
  • רספונסיביות - התאמה למסכי מובייל וטאבלט
  • העדר נתונים - הצגת הודעה מתאימה כאשר אין אנשי קשר

2.2 פונקציונליות חיפוש

רכיב תיאור התנהגות
שדה חיפוש תיבת טקסט בראש הדף חיפוש בזמן אמת בכל השדות
סינון החלת פילטר על הרשימה הצגת תוצאות מסוננות מיידית
איפוס חיפוש כפתור X / ריקון שדה החזרת כל אנשי הקשר לתצוגה

דרישות חיפוש:

  • חיפוש דינמי: עדכון תוצאות בזמן הקלדה
  • חיפוש רב-שדות: חיפוש בשם, טלפון, אימייל וכתובת
  • Case-insensitive: אי התחשבות באותיות גדולות/קטנות
  • Trim: התעלמות מרווחים מיותרים
  • הדגשת תוצאות: (אופציונלי) הדגשת טקסט התואם לחיפוש

2.3 יצירת איש קשר חדש (Create)

טופס הוספת איש קשר

שדות הטופס:

שדה סוג חובה/רשות ולידציה
שם מלא טקסט חובה מינימום 2 תווים
טלפון טקסט/טלפון חובה פורמט טלפון תקין
אימייל email רצוי פורמט אימייל תקין
כתובת textarea רשות -

דרישות פונקציונליות:

  • ולידציה בצד לקוח: בדיקת שדות לפני שליחה
  • הצגת שגיאות: הודעות ברורות מתחת לכל שדה
  • כפתור "שמור": שמירת איש הקשר והוספה לרשימה
  • כפתור "ביטול": סגירת הטופס מבלי לשמור
  • איפוס טופס: ניקוי כל השדות לאחר שמירה מוצלחת
  • הודעת הצלחה: אינדיקציה ויזואלית לשמירה מוצלחת

2.4 עריכת איש קשר (Update)

דרישות תצוגתיות:

  • פתיחת אותו טופס כמו ב"יצירה חדשה"
  • מילוי אוטומטי של כל השדות עם נתוני איש הקשר
  • כותרת ברורה: "עריכת איש קשר: [שם]"
  • כפתורי "שמור שינויים" ו"ביטול"

דרישות פונקציונליות:

  • שמירת שינויים תחליף את הנתונים הקיימים
  • ולידציה מלאה כמו ביצירת איש קשר חדש
  • עדכון מיידי של הרשימה לאחר שמירה
  • אפשרות לביטול העריכה מבלי לשמור

2.5 מחיקת איש קשר (Delete)

⚠️ אזהרה: מחיקה היא פעולה בלתי הפיכה!

דרישות UX למחיקה:

  • אישור מחיקה: חלון Confirm/Modal לפני מחיקה
  • הודעת אזהרה: "האם אתה בטוח שברצונך למחוק את [שם]?"
  • כפתורי אישור: "אישור מחיקה" (אדום) ו"ביטול" (אפור)
  • הודעת הצלחה: אינדיקציה שהמחיקה בוצעה בהצלחה
  • עדכון רשימה: הסרה מיידית של איש הקשר מהתצוגה

2.6 אחסון נתונים

דרישות אחסון:

  • Local Storage: שמירת נתונים בדפדפן
  • התמדה: נתונים נשמרים גם לאחר רענון הדף
  • ניהול State: שימוש ב-React State לניהול הנתונים
  • מפתחות ייחודיים: כל איש קשר מקבל ID ייחודי

3. דף Playground

3.1 מטרת הדף

דף Playground הוא דף הדגמה המציג מגוון אלמנטי HTML ורכיבי UI לצרכי:

  • בדיקת סלקטורים ב-Selenium/Cypress/Playwright
  • תרגול כתיבת בדיקות אוטומטיות
  • הדגמת אלמנטים שונים למטרות לימוד
  • בדיקות נגישות (Accessibility Testing)

אלמנטי טפסים

  • רשימות נפתחות (Select/Dropdown)
  • Text Area
  • Date Picker
🔘

כפתורים ולינקים

  • כפתורים רגילים (Buttons)
  • כפתורים מושבתים (Disabled)

  • כפתורי Submit
📊

אלמנטים מורכבים

  • רשימות (Lists: ul, ol)
  • כרטיסים (Cards)

3.3 דרישות תצוגתיות לדף Playground

ארגון הדף:

  • סקשנים מסודרים: כל קבוצת אלמנטים בסקשן נפרד עם כותרת
  • Labels ברורים: כל אלמנט מתויג בבירור
  • קוד לדוגמה: (אופציונלי) הצגת הקוד של כל אלמנט
  • ניווט פנימי: תפריט קפיצה לסקשנים השונים
  • רווחים נאותים: מרווחים ברורים בין אלמנטים

דרישות נגישות:

  • כל אלמנט טופס עם label מתאים
  • שימוש נכון ב-aria-labels
  • תמיכה בניווט מקלדת
  • ניגודיות צבעים תקינה

3.4 אינטראקטיביות (אופציונלי)

  • כפתורים שמדפיסים הודעות ל-console
  • טפסים שמציגים Alert עם הערכים שהוזנו
  • Toggle buttons שמשנים מצב
  • דוגמאות לאנימציות CSS פשוטות

4. ניווט באפליקציה

4.1 תפריט ניווט ראשי

Header / Navigation Bar

רכיבי התפריט:

  • לוגו/כותרת: "Testing Talks App" או שם דומה
  • קישור "Contacts": מוביל לדף ניהול אנשי קשר
  • קישור "Playground": מוביל לדף ה-Playground
  • קישור "Home": (אופציונלי) דף בית

דרישות ניווט:

  • React Router: שימוש ב-React Router לניתוב
  • Active State: הדגשת הדף הפעיל בתפריט
  • רספונסיבי: תפריט המבורגר במובייל
  • נגיש: ניתן לניווט באמצעות מקלדת

4.2 מסלולי ניתוב (Routes)

מסלול קומפוננטה תיאור
/ Home / Contacts דף ראשי (ברירת מחדל)
/contacts ContactsList רשימת אנשי קשר
/playground Playground דף הדגמת אלמנטים
/contacts/new ContactForm יצירת איש קשר חדש (אופציונלי)
/contacts/:id ContactDetails פרטי איש קשר (אופציונלי)

5.3 טיפוגרפיה

5.4 רספונסיביות

גודל מסך התנהגות
מובייל (< 768px) תצוגה חד-עמודתית, תפריט המבורגר, טפסים מלאי רוחב
טאבלט (768px - 1024px) תצוגה דו-עמודתית, תפריט מלא, גריד נקי
דסקטופ (> 1024px) תצוגה מלאה, ניצול אופטימלי של רוחב המסך

5.5 אנימציות ואפקטים

  • Hover Effects: שינוי צבע/צל בעת ריחוף על כפתורים
  • Transitions: מעברים חלקים בין מצבים (0.3s)
  • Loading States: ספינרים או skeleton screens בטעינה
  • Success/Error: הנפשות קלות להודעות

7. בדיקות ונגישות

7.1 אסטרטגיית בדיקה

האפליקציה משמשת כפלטפורמה לתרגול בדיקות, לכן חשוב שהיא תהיה ידידותית לבדיקה:

Data Attributes לבדיקות:

  • data-testid: זיהוי ייחודי לכל אלמנט משמעותי
  • id attributes: מזהים ייחודיים לאלמנטי טופס
  • name attributes: לכל שדה טופס
  • class names: עקביים וברורים
דוגמאות לסלקטורים
// כפתור הוספת איש קשר
<button data-testid="add-contact-btn" className="btn-primary">
  הוסף איש קשר
</button>

// שדה חיפוש
<input 
  data-testid="search-input" 
  id="contact-search"
  name="search"
  placeholder="חפש איש קשר..."
/>

// כפתור מחיקה
<button 
  data-testid={`delete-contact-${contact.id}`}
  className="btn-delete"
>
  מחק
</button>

7.2 תרחישי בדיקה מומלצים

  • יצירת איש קשר חדש עם כל השדות
  • יצירת איש קשר עם שדות חובה בלבד
  • ולידציה - ניסיון שמירה עם שדות חסרים
  • חיפוש איש קשר לפי שם/טלפון/אימייל
  • עריכת פרטי איש קשר קיים
  • מחיקת איש קשר עם אישור
  • ביטול מחיקה
  • בדיקת persistency - רענון דף

7.3 נגישות (A11y)

  • כל תמונה עם alt text
  • כל טופס עם <label> מתאים
  • סדר Tab לוגי
  • Focus visible על אלמנטים אינטראקטיביים
  • ניגודיות WCAG AA לפחות
  • הודעות שגיאה קריאות לקוראי מסך
